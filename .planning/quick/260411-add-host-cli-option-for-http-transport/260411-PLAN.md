`WebApplication.CreateBuilder(args)` already registers a default command-line configuration provider
for `Key=Value` style and `--Key Value` style that uses the FULL colon-separated key. To enable

`CreateBuilder(args)`, so it is the last-registered provider and wins:

```csharp
builder.Configuration.AddCommandLine(args, new Dictionary<string, string>
{
    { "--host", "Transport:Http:Host" },
    { "--port", "Transport:Http:Port" },
});
```

`AddCommandLine` is in namespace `Microsoft.Extensions.Configuration` (the using is already present in Program.cs:8).

Current stdio branch (Program.cs:77-101) has no pre-check. The reject pre-check must run BEFORE
`Host.CreateApplicationBuilder(args)` so stdio startup never begins with misapplied flags.

`Console.Error.WriteLine(...)` + `return 2;` (or any non-zero). Exit code 2 is conventional for
"bad usage" in POSIX tools; keep it distinct from 1 which startup exceptions already produce.
