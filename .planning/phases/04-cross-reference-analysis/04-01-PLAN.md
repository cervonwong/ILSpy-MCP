# Phase 4 Plan 1: Cross-Reference Analysis Domain, Infrastructure, and Use Cases

## Objective

Build the IL scanning infrastructure and domain layer for cross-reference analysis: find usages (call sites, field/property accesses), find implementors (interface/base class implementations), find dependencies (outward calls/refs), and find instantiations (newobj sites).

## Context

- @Domain/Services/IDecompilerService.cs — existing domain interface pattern
- @Domain/Services/IDisassemblyService.cs — separate service interface pattern (Phase 3 decision)
- @Infrastructure/Decompiler/ILSpyDisassemblyService.cs — adapter pattern using ICSharpCode.Decompiler
- @Application/UseCases/DisassembleTypeUseCase.cs — use case pattern with timeout/concurrency
- @TestTargets/Types/ServiceTypes.cs — cross-reference test targets (ServiceA calls ServiceB)
- @TestTargets/Types/InterfaceTypes.cs — interface implementation test targets

### Task 2: Infrastructure IL scanner implementation
type="auto"

Implement ICrossReferenceService using System.Reflection.Metadata IL scanning.

**Files to create:**
- `Infrastructure/Decompiler/ILSpyCrossReferenceService.cs` — adapter implementation

**Implementation approach:**
- Use MetadataReader + MethodBodyBlock.GetILReader() for IL scanning
- FindUsages: scan all method bodies for call/callvirt/ldfld/stfld/ldsfld/stsfld opcodes matching target member
- FindImplementors: use ITypeDefinition.DirectBaseTypes to find interface/base class implementations
- FindDependencies: scan a method/type's IL for outward call/field references
- FindInstantiations: scan all method bodies for newobj opcode matching target type
- Error handling follows existing patterns (TypeNotFoundException, AssemblyLoadException)

**Done criteria:**
- All 4 methods implemented with IL scanning
- Handles edge cases (abstract methods, generics, missing metadata)
