**3. Create `Infrastructure/Decompiler/ILSpySearchService.cs`** (per D-05, D-06, D-07):

Implement `ISearchService`. Follow the exact same class structure as `ILSpyCrossReferenceService`:
- Constructor takes `ILogger<ILSpySearchService>`, creates `DecompilerSettings` with `ThrowOnAssemblyResolveErrors = false, ShowXmlDocumentation = false`.
- Both methods wrap in `Task.Run(() => { ... })`.
- Create `CSharpDecompiler(assemblyPath.Value, _settings)` for type system access.
- Get `metadataFile` from `decompiler.TypeSystem.MainModule.MetadataFile`.
- Iterate `decompiler.TypeSystem.MainModule.TypeDefinitions`, skip types where `ParentModule != MainModule`.
- For each type, iterate `Methods`, skip if `MetadataToken.IsNil` or `RelativeVirtualAddress == 0`.
- Get method body via `metadataFile.GetMethodBody(methodDef.RelativeVirtualAddress)`, get `BlobReader` via `body.GetILReader()`.

**Duplicate the IL scanning static helpers** from `ILSpyCrossReferenceService` (lines 571-780): `ReadILOpCode`, `SkipOperand`, `GetOperandSize`, `IsTokenReferenceOpCode`, `GetTypeFullName`, `FormatMethodSignature`. These are small private static methods (~120 lines total). Duplication avoids coupling between services.

**For `SearchStringsAsync`** (SRCH-01):
- Compile regex once: `var regex = new Regex(regexPattern, RegexOptions.Compiled)`.
- In the IL scan loop, when `opCode == ILOpCode.Ldstr`: read 4-byte token via `ilReader.ReadInt32()`, convert to UserStringHandle via `MetadataTokens.UserStringHandle(token & 0x00FFFFFF)`, read string via `reader.GetUserString(handle)`, match against regex.
- Collect ALL matches into a full list first, then apply offset/limit pagination.
- Return `SearchResults<StringSearchResult>` with `TotalCount = allMatches.Count`, `Offset = offset`, `Limit = maxResults`, `Results = allMatches.Skip(offset).Take(maxResults).ToList()`.

**For `SearchConstantsAsync`** (SRCH-02):
- In the IL scan loop, check for `Ldc_i4_m1` through `Ldc_i4_8` (implicit values -1 to 8, no operand), `Ldc_i4_s` (ReadSByte), `Ldc_i4` (ReadInt32), `Ldc_i8` (ReadInt64).
- Compare extracted value to target `long value` parameter.
- Set `ConstantType` to "Int32" for `Ldc_i4*`/`Ldc_i4_s`, "Int64" for `Ldc_i8`.
- Do NOT handle `Ldc_r4`/`Ldc_r8` (float/double) — integer search only per D-07 "exact numeric value".
- Same pagination pattern as SearchStrings.

Handle exceptions: catch `BadImageFormatException` per method body (skip corrupted bodies, log warning), propagate `OperationCanceledException`. Wrap in `AssemblyLoadException` if `CSharpDecompiler` constructor throws.
