  <name>Task 1: Domain model and use case for project export</name>
  <files>Domain/Models/ProjectExportResult.cs, Application/UseCases/ExportProjectUseCase.cs</files>
  <action>
1. Create `Domain/Models/ProjectExportResult.cs`:
   - `ProjectExportResult` record with: `string OutputDirectory`, `string CsprojPath` (relative to output dir), `IReadOnlyList<string> SourceFiles` (relative paths of .cs files), `int TotalFileCount`, `IReadOnlyList<string> Warnings` (for partial failures per D-11)

2. Create `Application/UseCases/ExportProjectUseCase.cs`:
   - Constructor: `ITimeoutService`, `IConcurrencyLimiter`, `ILogger<ExportProjectUseCase>` (no IDecompilerService needed -- uses WholeProjectDecompiler directly)
   - Method: `Task<string> ExecuteAsync(string assemblyPath, string outputDirectory, string? namespaceFilter = null, int maxTypes = 500, CancellationToken cancellationToken = default)`
   - Implementation:
     a. Validate `assemblyPath` via `AssemblyPath.Create(assemblyPath)`
     b. **Directory validation (D-08, D-09):**
        - If directory exists AND has any entries (`Directory.EnumerateFileSystemEntries(outputDirectory).Any()`): throw `McpToolException("DIRECTORY_NOT_EMPTY", "Output directory is not empty: {outputDirectory}. Specify an empty or non-existent directory.")`
        - If directory doesn't exist: `Directory.CreateDirectory(outputDirectory)`
     c. Wrap the export in `_limiter.ExecuteAsync` + `_timeout.CreateTimeoutToken`
     d. Inside the limiter, run on `Task.Run` (WholeProjectDecompiler is CPU-bound):
        - Create `PEFile` from assembly path (using statement -- must outlive DecompileProject)
        - Create `UniversalAssemblyResolver(assemblyPath, throwOnError: false, targetFramework: null)`
        - If `namespaceFilter` is provided, create a subclass `NamespaceFilteredProjectDecompiler` (see below) that overrides type inclusion. If the `IncludeTypeWhenDecompilingProject` virtual method is not available in v10.0, fall back to full export and note the limitation.
        - Otherwise create `new WholeProjectDecompiler(resolver)`
        - Set `decompiler.Settings.ThrowOnAssemblyResolveErrors = false`
        - Call `decompiler.DecompileProject(peFile, outputDirectory, cancellationToken)`
        - Catch any exception during export, add to warnings list (D-11 partial failure)
     e. After export, enumerate output files:
        - Find .csproj: `Directory.GetFiles(outputDirectory, "*.csproj", SearchOption.TopDirectoryOnly).FirstOrDefault()`
        - Find .cs files: `Directory.GetFiles(outputDirectory, "*.cs", SearchOption.AllDirectories)` and convert to relative paths
        - Build `ProjectExportResult`
     f. Format output string:
        ```
        Project exported to: {outputDirectory}
        Project file: {csprojRelativePath}
        Total files: {count}

        Source files:
          {relative/path/File1.cs}
          {relative/path/File2.cs}
          ...

        {If warnings:}
        Warnings:
          - {warning1}
          - {warning2}
        ```

   - Create `NamespaceFilteredProjectDecompiler` as a private nested class or internal class in the same file:
     ```csharp
     internal class NamespaceFilteredProjectDecompiler : WholeProjectDecompiler
     {
         private readonly string _namespace;
         public NamespaceFilteredProjectDecompiler(string ns, IAssemblyResolver resolver)
             : base(resolver) { _namespace = ns; }
         // Override IncludeTypeWhenDecompilingProject if available
         // If not available in v10.0, document limitation and skip namespace filtering for export
     }
     ```
     If `IncludeTypeWhenDecompilingProject` doesn't exist or isn't overridable, simply do a full export and log a warning that namespace filtering is not supported for project export. The namespace parameter becomes a no-op with a warning rather than breaking.

   - Standard error handling: OperationCanceledException -> TimeoutException, rethrow DomainException
  </action>
