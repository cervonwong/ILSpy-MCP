  <name>Task 1: Domain model and use case for namespace decompilation</name>
  <files>Domain/Models/NamespaceTypeSummary.cs, Application/UseCases/DecompileNamespaceUseCase.cs</files>
  <action>
1. Create `Domain/Models/NamespaceTypeSummary.cs` with a result model:
   - `NamespaceTypeSummary` record with: `string Namespace`, `IReadOnlyList<TypeSummaryEntry> Types`, `int TotalTypeCount`
   - `TypeSummaryEntry` record with: `string FullName`, `string ShortName`, `TypeKind Kind`, `string? BaseType` (first base type or null), `int MethodCount`, `int PropertyCount`, `int FieldCount`, `IReadOnlyList<string> PublicMethodSignatures` (formatted like `string ToString()`), `IReadOnlyList<TypeSummaryEntry>? NestedTypes`

2. Create `Application/UseCases/DecompileNamespaceUseCase.cs`:
   - Constructor: `IDecompilerService`, `ITimeoutService`, `IConcurrencyLimiter`, `ILogger<DecompileNamespaceUseCase>`
   - Method: `Task<string> ExecuteAsync(string assemblyPath, string namespaceName, int maxTypes = 200, CancellationToken cancellationToken = default)`
   - Implementation:
     a. Create `AssemblyPath.Create(assemblyPath)`
     b. Call `_decompiler.ListTypesAsync(assembly, namespaceName)` to get types
     c. **CRITICAL**: Post-filter to exact namespace match: `type.Namespace == namespaceName` (ListTypesAsync uses Contains matching which includes sub-namespaces)
     d. If no types match, throw `NamespaceNotFoundException` (create in Domain/Errors if needed, or use existing pattern). Error code: `NAMESPACE_NOT_FOUND`, message: `"Namespace '{namespaceName}' not found. Use list_namespaces to see available namespaces."`
     e. Apply `maxTypes` limit after filtering
     f. Sort by kind order: Interface=0, Enum=1, Struct=2, Class=3, Delegate=4, Unknown=5, then alphabetically by FullName within each group (per D-03)
     g. Separate top-level types from nested types. A type is nested if its FullName contains `+` (e.g., `Outer+Inner`). Group nested types under their parent.
     h. Format output as a readable string (see format below)
   - Wrap in `_limiter.ExecuteAsync` + `_timeout.CreateTimeoutToken` per established pattern
   - Use standard error handling pattern (OperationCanceledException -> TimeoutException, DomainException rethrow)

   Output format (per D-01, D-02, D-03):
   ```
   Namespace: {namespaceName} ({N} types)

   Interfaces:
     interface IShape
       Methods: 1 | Properties: 0 | Fields: 0
       double CalculateArea()

   Classes:
     class Circle : Shape
       Methods: 2 | Properties: 1 | Fields: 1
       double CalculateArea()
       string ToString()
       Nested types:
         class CircleComparer
           Methods: 1 | Properties: 0 | Fields: 0
   ```

   For public method signatures, format as: `{ReturnType} {Name}({Param1Type} {Param1Name}, ...)`. Only include public methods (Accessibility == Public). Include constructors in the method count but not in the signature listing.
  </action>
