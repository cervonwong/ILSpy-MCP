Specific parameters per use case:
1. **GetAssemblyMetadataUseCase**: `ExecuteAsync(string assemblyPath, CancellationToken)` -> returns formatted string with metadata fields and references table
2. **GetAssemblyAttributesUseCase**: `ExecuteAsync(string assemblyPath, CancellationToken)` -> returns formatted attribute list
3. **GetTypeAttributesUseCase**: `ExecuteAsync(string assemblyPath, string typeName, CancellationToken)` -> returns formatted attribute list
4. **GetMemberAttributesUseCase**: `ExecuteAsync(string assemblyPath, string typeName, string memberName, CancellationToken)` -> returns formatted attribute list
5. **ListEmbeddedResourcesUseCase**: `ExecuteAsync(string assemblyPath, CancellationToken)` -> returns formatted resource catalog
6. **ExtractResourceUseCase**: `ExecuteAsync(string assemblyPath, string resourceName, int? offset, int? limit, CancellationToken)` -> returns formatted resource content
7. **FindCompilerGeneratedTypesUseCase**: `ExecuteAsync(string assemblyPath, CancellationToken)` -> returns formatted list with parent context

Each use case formats the service result into a human-readable string (same pattern as existing use cases that return `string`). Format assembly metadata as a clear text block with field labels. Format lists with one item per line. Include headers.

**7 MCP Tools** — Each follows DisassembleTypeTool pattern exactly:
- `[McpServerToolType]` class attribute
- Constructor injects use case + `ILogger<T>`
- `[McpServerTool(Name = "tool_name")]` and `[Description("...")]` on `ExecuteAsync`
- Parameters with `[Description("...")]` attributes
- Catch `TypeNotFoundException` -> `McpToolException("TYPE_NOT_FOUND", ...)`, `AssemblyLoadException` -> `McpToolException("ASSEMBLY_LOAD_FAILED", ...)`, `MethodNotFoundException` -> `McpToolException("MEMBER_NOT_FOUND", ...)`, `TimeoutException` -> `McpToolException("TIMEOUT", ...)`, `OperationCanceledException` -> `McpToolException("CANCELLED", ...)`, `Exception` -> `McpToolException("INTERNAL_ERROR", ...)`

Tool names and parameters (per D-01, D-02, D-06):
1. **get_assembly_metadata**: `assemblyPath` only. Description: "Get assembly metadata including target framework, runtime version, PE bitness, strong name, entry point, and all referenced assemblies"
2. **get_assembly_attributes**: `assemblyPath` only. Description: "List all custom attributes declared on an assembly with their constructor arguments and named properties"
3. **get_type_attributes**: `assemblyPath` + `typeName`. Description: "List all custom attributes declared on a type with their constructor arguments and named properties"
4. **get_member_attributes**: `assemblyPath` + `typeName` + `memberName`. Description: "List all custom attributes on a type member (method, property, field, event) with their constructor arguments"
5. **list_embedded_resources**: `assemblyPath` only. Description: "List all embedded resources in an assembly with name, type, size, and visibility"
6. **extract_resource**: `assemblyPath` + `resourceName` + optional `offset` (int, default null) + optional `limit` (int, default null). Description: "Extract embedded resource content. Returns text inline or binary as base64. Use offset and limit for paginated binary extraction."
7. **find_compiler_generated_types**: `assemblyPath` only. Description: "Find compiler-generated types (async state machines, display classes, closures, iterators) with their parent method and type context"
