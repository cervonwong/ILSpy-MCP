**StringAttributeTypeProvider.cs** â€” Implements `ICustomAttributeTypeProvider<string>` for decoding attribute constructor arguments (per research Pattern 4). This avoids hand-rolling ECMA-335 blob parsing:
```csharp
using System.Reflection.Metadata;

namespace ILSpy.Mcp.Infrastructure.Decompiler;

internal sealed class StringAttributeTypeProvider : ICustomAttributeTypeProvider<string>
{
    public string GetPrimitiveType(PrimitiveTypeCode typeCode) => typeCode.ToString();
    public string GetTypeFromDefinition(MetadataReader reader, TypeDefinitionHandle handle, byte rawTypeKind)
    {
        var typeDef = reader.GetTypeDefinition(handle);
        var ns = reader.GetString(typeDef.Namespace);
        var name = reader.GetString(typeDef.Name);
        return string.IsNullOrEmpty(ns) ? name : $"{ns}.{name}";
    }
    public string GetTypeFromReference(MetadataReader reader, TypeReferenceHandle handle, byte rawTypeKind)
    {
        var typeRef = reader.GetTypeReference(handle);
        var ns = reader.GetString(typeRef.Namespace);
        var name = reader.GetString(typeRef.Name);
        return string.IsNullOrEmpty(ns) ? name : $"{ns}.{name}";
    }
    public string GetTypeFromSerializedName(string name) => name;
    public string GetSZArrayType(string elementType) => $"{elementType}[]";
    public string GetSystemType() => "System.Type";
    public bool IsSystemType(string type) => type == "System.Type" || type == "Type";
    public PrimitiveTypeCode GetUnderlyingEnumType(string type) => PrimitiveTypeCode.Int32; // safe default for display
}
```

**ILSpyAssemblyInspectionService.cs** â€” Full implementation of `IAssemblyInspectionService`. Per D-12, uses S.R.Metadata APIs + ICSharpCode.Decompiler type system. Follow the anti-pattern guidance from research: do NOT create both CSharpDecompiler and separate PEFile â€” use decompiler's MetadataFile for both MetadataReader and PEHeaders access.

Key implementation details for each method:

1. **GetAssemblyMetadataAsync** (META-01 + META-02, per D-03):
   - Create `CSharpDecompiler` with default `DecompilerSettings`
   - Access `decompiler.TypeSystem.MainModule.MetadataFile` for MetadataReader and PEHeaders
   - Read `AssemblyDefinition` for name, version, culture, publicKey
   - Read `PEHeaders.CoffHeader.Machine` for PE bitness: `Machine.I386` with `CorFlags.Requires32Bit` = "x86", `Machine.I386` without = "AnyCPU", `Machine.Amd64` = "x64", `Machine.Arm64` = "ARM64"
   - Read `TargetFrameworkAttribute` from assembly custom attributes for target framework string
   - Read `CorHeader.EntryPointTokenOrRelativeVirtualAddress` for entry point, resolve to `Namespace.Type.Method` format
   - Read `CorHeader.MajorRuntimeVersion.MinorRuntimeVersion` for runtime version
   - Iterate `reader.AssemblyReferences` to build `IReadOnlyList<AssemblyReferenceInfo>` with name, version, culture, public key token
   - For public key token: read `asmRef.PublicKeyOrToken` blob bytes, format as lowercase hex string (e.g., "b77a5c561934e089")
   - Strong name: if `assemblyDef.PublicKey` is not nil, format as hex

2. **GetAssemblyAttributesAsync** (META-03, per D-04/D-06):
   - Use `reader.GetAssemblyDefinition().GetCustomAttributes()` to get attribute handles
   - For each `CustomAttributeHandle`, get `CustomAttribute`, decode constructor via `attr.DecodeValue(new StringAttributeTypeProvider())`
   - Build `AttributeInfo` with full type name from `attr.Constructor` (resolve via `MemberReferenceHandle` or `MethodDefinitionHandle`), `FixedArguments` mapped to string values, `NamedArguments` mapped to name->value dict
   - Return ALL attributes unfiltered (per pitfall 6 guidance)

3. **GetTypeAttributesAsync** (META-04, per D-04/D-06):
   - Find type by `TypeName` using decompiler's `TypeSystem.FindType()` or iterate `reader.TypeDefinitions` matching full name
   - Get `TypeDefinition.GetCustomAttributes()`, decode same as assembly attributes
   - Throw `TypeNotFoundException` if type not found

4. **GetMemberAttributesAsync** (META-04, per D-04/D-06):
   - Find type first (same as GetTypeAttributesAsync)
   - Search ALL member kinds: methods, fields, properties, events matching `memberName` (per pitfall 5)
   - For properties: check both the property itself and its getter/setter methods
   - Return attributes from ALL matching members, include member kind in attribute type prefix if ambiguous
   - Throw `TypeNotFoundException` for bad type, `MethodNotFoundException` for member not found (reuse existing exception, maps to MEMBER_NOT_FOUND)

5. **ListEmbeddedResourcesAsync** (RES-01, per D-07):
   - Use ICSharpCode.Decompiler's `metadataFile.Resources` collection (higher level than raw S.R.Metadata)
   - For each resource: name, `ResourceType` ("Embedded" or "Linked"), size (from stream length or raw section data length), visibility from `ManifestResourceAttributes`
   - For embedded resources only: get size by reading the 4-byte Int32 length prefix from the resource data section

6. **ExtractResourceAsync** (RES-02, per D-08):
   - Find the resource by name in `metadataFile.Resources`
   - If not found, throw an appropriate exception
   - Open stream via `resource.TryOpenStream()`, read all bytes
   - Determine content type: try UTF-8 decoding â€” if valid text with no control chars (except \r\n\t), return as "text" inline; otherwise "binary" as base64
   - Apply `offset` and `limit` on raw bytes BEFORE encoding (per research recommendation): slice the byte array, then either UTF-8 decode or base64 encode the slice
   - Set `TotalSize` to full resource size, `Offset` and `Length` to actual slice parameters

7. **FindCompilerGeneratedTypesAsync** (TYPE-01 + TYPE-02, per D-09/D-10):
   - Use `decompiler.TypeSystem.MainModule.TypeDefinitions` to iterate ALL types
   - For each type, check if it's compiler-generated using BOTH naming conventions AND `CompilerGeneratedAttribute`:
     - Contains `DisplayClass` or `AnonStorey` or starts with `<>c__` => "DisplayClass"
     - Contains `d__` (and is nested) => check if implements `IAsyncStateMachine` for "AsyncStateMachine", else if implements `IEnumerator` for "Iterator"
     - Name is `<>c` (static lambda cache) => "Closure"
     - Has `CompilerGeneratedAttribute` but doesn't match above => "CompilerGenerated"
   - Extract parent method from naming convention: text between `<` and `>` delimiters (per research Pattern 6)
   - Set `ParentType` to the declaring type's full name
   - Return only nested/compiler-generated types, not user-defined nested types
